# DMG Installer Assets

This folder contains the DMG installer window assets exported from the Figma node:

`https://www.figma.com/design/zRe1li9DBtBHKongnfJUgE/macos-menubar-app?node-id=2001-1206`

Files:

- `assets/figma-reference.png`: full 540x340 Figma reference export.
- `assets/figma-arrow.svg`: exported arrow vector from Figma.
- `assets/figma-appicon.svg`: exported app icon asset from Figma.
- `generated/dmg-background.png`: usable Finder background, regenerated by `generate-background.py`.
- `generated/dmg-background@2x.png`: retina-sized version for external tooling.
- `create-dmg.sh`: builds `Artifacts/OnePlus-Buds-Menu.dmg` and positions the real app icon on the left and `/Applications` alias on the right.

The DMG uses the real `.app` bundle and `/Applications` alias as Finder icons. Finder does not expose a stable setting for icon-label text color in DMG icon view, so the label color is system-controlled; the background is kept dark enough for Finder to render readable light labels.

Usage:

```sh
Packaging/DMG/create-dmg.sh
```

Or pass a specific app bundle:

```sh
Packaging/DMG/create-dmg.sh "/path/to/OnePlus Buds Menu.app"
```

`create-dmg.sh` only creates the disk image. For a publishable GitHub Release,
use `Packaging/release.sh` from the repository root. It archives a universal
Release build, verifies the app signature and entitlements, signs and notarizes
the DMG, staples the ticket, runs Gatekeeper checks, and writes a SHA-256 file.
